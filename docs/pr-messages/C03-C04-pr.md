# PR Implementation Report: C03 + C04

## Summary

C03 adds `crate_smoke_aud15_scheduler_yields_without_proximity_clamp`, a new smoke test that
proves the smart scheduler — not `clamp_velocity_for_proximity` — is responsible for reducing a
managed vehicle's `commanded_velocity` on a conflict lane pair. It also fixes the
`south_straight_lane_vehicle` helper in `vehicle.rs`, which previously left `nominal_velocity` at
the `id%3`-assigned level rather than the caller-supplied speed; this could produce vacuous
`saw_slowdown` results in any future test that reused that helper.

C04 adds a `LANE_CAPACITY = 8` congestion cap in `SpawnSystem::try_spawn` that blocks new
spawns into a lane already holding 8 or more active vehicles, and adds
`crate_smoke_aud16_aud17_sustained_no_overlap_no_lane_overflow`, a 60-second (3 600-frame)
simulation that asserts no two vehicles overlap and no lane exceeds 8 vehicles simultaneously.

Covers: **AUD-15** (scheduler yield proof), **AUD-16** (sustained no-overlap), **AUD-17**
(congestion cap), **REQ-3** (smart algorithm), **REQ-7** (velocity levels), **REQ-9** (collision
avoidance).

## Key Changes

- **`src/vehicle.rs` — `south_straight_lane_vehicle` helper**: added `vehicle.nominal_velocity = speed;` so the helper assigns the caller-supplied speed as the nominal, preventing vacuous `saw_slowdown` results from id%3 level mismatch.
- **`src/spawn.rs` — congestion cap**: added `const LANE_CAPACITY: usize = 8` and a pre-spawn count check in `try_spawn`; returns `None` when a lane already holds ≥ 8 active vehicles.
- **`tests/smoke.rs` — AUD-15 test** (`crate_smoke_aud15_scheduler_yields_without_proximity_clamp`): builds two managed vehicles on South+Straight / East+Straight (confirmed conflict pair), calls only `SmartController::update()` (no position advance, no proximity clamp), then asserts positions are unchanged and `managed_scheduler_yielded` returns `true`.
- **`tests/smoke.rs` — AUD-16/17 test** (`crate_smoke_aud16_aud17_sustained_no_overlap_no_lane_overflow`): seeds one vehicle per approach, runs 3 600 frames calling `spawn_random` each frame, asserts gap ≥ `VEHICLE_LENGTH * 0.9` for every vehicle pair and lane count ≤ 8 every frame.
- **`docs/ticket-tracker.md`**: C03 and C04 set to ✅ Done.

## Technical Decisions

- **Bypass clamp via direct `SmartController::update()` call (AUD-15)**: Rather than adding a test-only flag or mock, the AUD-15 test builds `Vehicle` structs directly (without `SpawnSystem`) and calls `smart.update()` alone. This avoids `clamp_velocity_for_proximity` entirely and proves causality: position is unchanged, yet `commanded_velocity` dropped — the scheduler did it.
- **Entry-sequence priming via two-phase update (AUD-15)**: `entry_sequence` is private to `SmartController`. The test registers the South vehicle first (single-element slice), then adds the East vehicle and calls `update()` again. This uses the existing public API to prime sequence numbers (0 and 1) without exposing internals.
- **Congestion cap in `try_spawn` only (C04)**: The cap counts non-Done vehicles on `req.lane_id` before acquiring a slot. This is the single gate; no additional state is needed. The count uses the same `VehicleState::Done` guard used throughout the spawn module.
- **`LANE_CAPACITY` as a module-level const in `spawn.rs`**: Keeping it local avoids touching `config.rs` (a Track A file). If it needs to be tunable from outside, a later ticket can promote it.
